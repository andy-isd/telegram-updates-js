# telegram-updates-js

Кастомна система для виклику власних вебхуків у відповідь на нові повідомлення в Telegram-каналах. Кожен користувач може налаштувати під себе: які канали слухати, які вебхуки викликати і за якими умовами.

## Мета продукту

- Слухати одночасно кілька Telegram-каналів через gramJS (MTProto, не Bot API)
- Викликати HTTP-вебхуки при появі нових повідомлень (з фільтрацією за ключовими словами, типом, відправником)
- Адмінка для керування каналами, вебхуками та перегляду статистики
- Авторизація в адмінці (JWT або сесії)

## Поточний стан

Реалізовано базовий listener одного каналу (`tgupdates.js`):
- gramJS підключення через `StringSession`, збережену у `storage/session.dat`
- Авторизація через `client.start()` з інтерактивним введенням коду
- Обробник `NewMessage` зберігає події у JSON-файли в `storage/<channelUsername>/`
- Запуск як daemon через PM2 (`ecosystem.config.js`)

## Стек

- **Runtime**: Node.js
- **Telegram**: `telegram` (gramJS) — MTProto, не Bot API
- **Process manager**: PM2
- **Env**: dotenv (`dotenv` пакет, `.env` файл)

## Змінні середовища (`.env`)

```
TELEGRAM_API_ID=       # з my.telegram.org
TELEGRAM_API_HASH=     # з my.telegram.org
PHONE_NUMBER=          # номер акаунта у форматі +380...
CHANNEL_USERNAME=      # username або ID каналу
```

## Запуск

```bash
# Розробка (інтерактивний, потрібен для першої авторизації)
node tgupdates.js

# Production через PM2
npm run pm2:start
npm run pm2:stop
```

Логи PM2: `logs/server.log`, `logs/server.error.log`

## Відомі нюанси

- **Перша авторизація** завжди інтерактивна: gramJS запитує код із Telegram app. Запускати `node tgupdates.js` вручну, не через PM2.
- **FLOOD помилка**: якщо Telegram повертає `FLOOD` з таймаутом — чекати вказану кількість секунд перед повторним запуском.
- **Втрата сесії**: якщо підключення з існуючою сесією падає — `session.dat` видаляється автоматично, потрібна повторна авторизація.
- **`onError` у `client.start()`**: не повертає `false`, через що gramJS може повторити запит коду — це відома поведінка, яку треба виправити.
